Adding New Resources

This guide explains how to add new resources to your Terraform configuration.

Adding Resources Step by Step

Step 1: Plan Your Resource

Before adding a resource, consider:

  • What type of resource is it? (database, schema, table, view, etc.)
  • What are its dependencies?
  • What provider does it need?
  • What privileges need to be granted?

Step 2: Add the Resource Definition

Add the resource to main.tf following existing patterns.

Step 3: Plan and Validate

terraform validate
terraform plan

Step 4: Apply

terraform apply

Adding a New Database

Example

Add to the database_map in the ALL_DATABASES module:

module "ALL_DATABASES" {
  source = "./modules/snowflake/database"
  
  database_map = {
    # ... existing databases ...
    "NEW_DATABASE" = {
      comment                     = "Description of the database"
      data_retention_time_in_days = 1
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.ANALYST.name
          privileges        = ["USAGE"]
        }
      ]
    }
  }
}

Considerations

  • Database names must be unique
  • Choose appropriate retention period
  • Grants are handled automatically via role_privileges

Adding a New Schema

Example

Add to the schema_map in the ALL_SCHEMAS module:

module "ALL_SCHEMAS" {
  source = "./modules/snowflake/schema"
  
  schema_map = {
    # ... existing schemas ...
    "NEW_SCHEMA" = {
      database_name       = module.ALL_DATABASES.DATABASES.NEW_DATABASE.name
      with_managed_access = false
      is_transient        = false
      comment             = "Description of the schema"
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.ANALYST.name
          privileges        = ["USAGE"]
        }
      ]
    }
  }
}

Considerations

  • Schema must reference an existing database (use module output)
  • Consider with_managed_access for security
  • Consider is_transient for temporary data
  • Grants are handled automatically via role_privileges

Adding a New Table

Example

Add to the table_map in the ALL_TABLES module:

module "ALL_TABLES" {
  source = "./modules/snowflake/tables"
  
  table_map = {
    # ... existing tables ...
    "NEW_TABLE" = {
      database_name = module.ALL_DATABASES.DATABASES.NEW_DATABASE.name
      schema_name   = module.ALL_SCHEMAS.SCHEMAS.NEW_SCHEMA.name
      comment       = "Description of the table"
      columns = [
        {
          name     = "ID"
          type     = "NUMBER(38,0)"
          nullable = false
        },
        {
          name = "NAME"
          type = "VARCHAR(255)"
        }
      ]
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.ANALYST.name
          privileges        = ["SELECT"]
        }
      ]
    }
  }
}

Considerations

  • Table must reference existing database and schema (use module outputs)
  • Choose appropriate data types
  • Consider nullable constraints
  • Add comments for documentation
  • Grants are handled automatically via role_privileges

Adding a New View

Example

Add to the view_map in the ALL_VIEWS module:

module "ALL_VIEWS" {
  source = "./modules/snowflake/views"
  
  view_map = {
    # ... existing views ...
    "NEW_VIEW" = {
      database_name = module.ALL_DATABASES.DATABASES.NEW_DATABASE.name
      schema_name   = module.ALL_SCHEMAS.SCHEMAS.NEW_SCHEMA.name
      statement     = <<-SQL
        SELECT 
          ID,
          NAME
        FROM ${module.ALL_TABLES.TABLES.NEW_TABLE.name}
        WHERE ACTIVE = TRUE;
      SQL
      comment       = "Description of the view"
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.ANALYST.name
          privileges        = ["SELECT"]
        }
      ]
    }
  }
}

Considerations

  • View must reference existing database and schema (use module outputs)
  • SQL statement must be valid
  • Reference tables using module outputs when possible
  • Grants are handled automatically via role_privileges

Note on Grants

All grants are now handled automatically within modules! You no longer need separate grant resources. Simply include role_privileges in the module configuration for databases, schemas, tables, views, and warehouses.

Adding Resources to Modules

Adding a Role

Edit the role_map in the ALL_ROLES module:

module "ALL_ROLES" {
  # ... existing configuration ...
  
  role_map = {
    # ... existing roles ...
    "NEW_ROLE" = {
      parent_role_name = "SYSADMIN"
      comment          = "New role description"
    }
  }
}

Adding a User

Edit the user_map in the ALL_USERS module:

module "ALL_USERS" {
  # ... existing configuration ...
  
  user_map = {
    # ... existing users ...
    "NEW_USER" = {
      first_name  = "New"
      last_name   = "User"
      email       = "new.user@example.com"
      user_type   = "person"
      roles_to_assign = [module.ALL_ROLES.ROLES.NEW_ROLE.name]
    }
  }
}

Adding a Warehouse

Edit the warehouse_map in the ALL_WAREHOUSES module:

module "ALL_WAREHOUSES" {
  # ... existing configuration ...
  
  warehouse_map = {
    # ... existing warehouses ...
    "NEW_WAREHOUSE" = {
      warehouse_type       = "STANDARD"
      warehouse_generation = "2"
      warehouse_size       = "SMALL"
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.NEW_ROLE.name
          privileges        = ["USAGE"]
        }
      ]
    }
  }
}

Complete Example: Adding a New Data Domain

This example shows adding a complete data domain using modules (database, schema, table, view, with automatic grants):

1. Add Database

module "ALL_DATABASES" {
  source = "./modules/snowflake/database"
  
  database_map = {
    # ... existing databases ...
    "SALES_DB" = {
      comment                     = "Sales database"
      data_retention_time_in_days = 30
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.SALES.name
          privileges        = ["USAGE"]
        }
      ]
    }
  }
}

2. Add Schema

module "ALL_SCHEMAS" {
  source = "./modules/snowflake/schema"
  
  schema_map = {
    # ... existing schemas ...
    "ORDERS" = {
      database_name       = module.ALL_DATABASES.DATABASES.SALES_DB.name
      with_managed_access = false
      is_transient        = false
      comment             = "Orders schema"
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.SALES.name
          privileges        = ["USAGE"]
        }
      ]
    }
  }
}

3. Add Table

module "ALL_TABLES" {
  source = "./modules/snowflake/tables"
  
  table_map = {
    # ... existing tables ...
    "ORDER_ITEMS" = {
      database_name = module.ALL_DATABASES.DATABASES.SALES_DB.name
      schema_name   = module.ALL_SCHEMAS.SCHEMAS.ORDERS.name
      comment       = "Order items table"
      columns = [
        {
          name     = "ORDER_ITEM_ID"
          type     = "NUMBER(38,0)"
          nullable = false
        },
        {
          name     = "ORDER_ID"
          type     = "NUMBER(38,0)"
          nullable = false
        },
        {
          name     = "PRODUCT_ID"
          type     = "NUMBER(38,0)"
          nullable = false
        },
        {
          name     = "QUANTITY"
          type     = "NUMBER(10,0)"
          nullable = false
        },
        {
          name     = "PRICE"
          type     = "NUMBER(10,2)"
          nullable = false
        }
      ]
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.SALES.name
          privileges        = ["SELECT", "INSERT", "UPDATE"]
        }
      ]
    }
  }
}

4. Add View

module "ALL_VIEWS" {
  source = "./modules/snowflake/views"
  
  view_map = {
    # ... existing views ...
    "ORDER_SUMMARY" = {
      database_name = module.ALL_DATABASES.DATABASES.SALES_DB.name
      schema_name   = module.ALL_SCHEMAS.SCHEMAS.ORDERS.name
      statement     = <<-SQL
        SELECT 
          ORDER_ID,
          COUNT(*) AS ITEM_COUNT,
          SUM(PRICE * QUANTITY) AS TOTAL_AMOUNT
        FROM ${module.ALL_TABLES.TABLES.ORDER_ITEMS.name}
        GROUP BY ORDER_ID;
      SQL
      comment       = "Order summary view"
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.SALES.name
          privileges        = ["SELECT"]
        }
      ]
    }
  }
}

Note: All grants are automatically handled by the modules - no separate grant resources needed!

Importing Existing Resources

If a resource already exists in Snowflake, import it:

# Import database
terraform import snowflake_database.EXISTING_DB "EXISTING_DB"

# Import schema
terraform import snowflake_schema.EXISTING_SCHEMA "DB.SCHEMA"

# Import table
terraform import snowflake_table.EXISTING_TABLE "DB.SCHEMA.TABLE"

# Import view
terraform import snowflake_view.EXISTING_VIEW "DB.SCHEMA.VIEW"

Then add the resource definition to main.tf matching the imported resource.

Best Practices

  1. Follow Naming Conventions: Use consistent naming patterns
  2. Add Comments: Document the purpose of each resource
  3. Check Dependencies: Ensure dependencies are correct
  4. Use Fully Qualified Names: Use fully_qualified_name for grants
  5. Plan Before Apply: Always run terraform plan first
  6. Test Incrementally: Add resources one at a time and test
  7. Version Control: Commit changes to version control

Troubleshooting

Resource Already Exists

Error:

Error: 2003: Object already exists

Solution:

  • Import the existing resource
  • Or remove it from Snowflake first
  • Or use a different name

Dependency Error

Error:

Error: Resource depends on resource that doesn't exist

Solution:

  • Check that dependencies are created first
  • Add explicit depends_on if needed
  • Verify resource names are correct

Invalid Reference

Error:

Error: Reference to undeclared resource

Solution:

  • Verify resource names match exactly
  • Check that modules are referenced correctly
  • Ensure resources are defined before use
