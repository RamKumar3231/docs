Database Module

The Database module manages Snowflake databases with automatic privilege grant management.

Overview

The Database module (modules/snowflake/database) creates Snowflake databases and automatically grants privileges to specified roles. This module provides a complete solution for database management including access control.

Module Location

modules/snowflake/database/
├── main.tf        # Database resource definitions and grants
├── variables.tf   # Input variables
└── outputs.tf     # Output values

Usage

Basic Database

module "ALL_DATABASES" {
  source = "./modules/snowflake/database"
  
  providers = {
    snowflake = snowflake.snowflake_sysadmin
  }
  
  database_map = {
    "MY_DATABASE" = {
      comment                     = "My database"
      data_retention_time_in_days = 1
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.ANALYST.name
          privileges        = ["USAGE"]
        }
      ]
    }
  }
}

Database with Multiple Role Grants

module "ALL_DATABASES" {

Input Variables

database_map

Type: map(object)
Required: Yes
Description: Map of database names to database configurations

Database Configuration Object

FieldTypeRequiredDefaultDescription
commentstringNo""Comment describing the database
data_retention_time_in_daysnumberNo1Number of days to retain data (Time Travel)
role_privilegeslist(object)Yes-List of role privilege grants
with_grant_optionboolNofalseWhether grants include grant option

Role Privilege Object

FieldTypeRequiredDescription
account_role_namestringYesName of the role receiving privileges
privilegeslist(string)YesList of privileges: “USAGE”, “MODIFY”, “CREATE SCHEMA”, “MONITOR”, “OWNERSHIP”, “ALL PRIVILEGES”

Resources Created

snowflake_database.DATABASES

Creates databases using for_each over the database_map.

Properties:

  • name: Database name (from map key)
  • comment: Database comment
  • data_retention_time_in_days: Time Travel retention period

snowflake_grant_privileges_to_account_role.DATABASE_GRANT

Automatically grants database privileges to roles based on the role_privileges configuration.

Properties:

  • account_role_name: Role receiving privileges
  • privileges: List of privileges
  • on_account_object.object_type: “DATABASE”
  • on_account_object.object_name: Database name
  • with_grant_option: Whether grant includes grant option

Dependencies:

  • Depends on snowflake_database.DATABASES to ensure databases exist before granting

Outputs

DATABASES

Type: map(object)

A map of all created database resources, keyed by database name.

Example Output:

{
  "ANALYTICS_DB" = {
    name    = "ANALYTICS_DB"
    comment = "Analytics database"
    # ... other attributes
  }
}

Usage:

# Reference a database in schema module
database_name = module.ALL_DATABASES.DATABASES.ANALYTICS_DB.name

Available Privileges

  • USAGE: Use the database (required to access schemas/tables)
  • MODIFY: Modify database settings
  • CREATE SCHEMA: Create schemas in the database
  • MONITOR: View database usage and metrics
  • OWNERSHIP: Full ownership of the database
  • ALL PRIVILEGES: All of the above

Data Retention (Time Travel)

The data_retention_time_in_days property controls Time Travel retention:

  • 1 day: Default, minimum retention
  • Up to 90 days: For Standard accounts
  • Up to 1 year: For Enterprise accounts (with additional cost)

Important:

  • Higher retention = higher storage costs
  • Time Travel allows querying historical data
  • Can restore dropped objects within retention period

Provider Requirements

The module requires the snowflake_sysadmin provider alias because:

  • Creating databases requires SYSADMIN privileges
  • Granting database privileges requires SYSADMIN privileges

Dependencies

Internal Dependencies

  • snowflake_grant_privileges_to_account_role.DATABASE_GRANT depends on snowflake_database.DATABASES

External Dependencies

  • Roles Module: Databases reference roles from the roles module for privilege grants

Best Practices

  1. Descriptive Names: Use clear, descriptive database names
  2. Comments: Always add comments explaining database purpose
  3. Retention Policy: Set appropriate retention based on needs and costs
  4. Access Control: Grant USAGE to roles that need access
  5. Minimal Privileges: Grant only necessary privileges (principle of least privilege)
  6. Grant Option: Use with_grant_option = false unless delegation is needed

Troubleshooting

Database Already Exists

Error:

Error: 2003: Object already exists

Solution:

  • Import existing database: terraform import module.ALL_DATABASES.snowflake_database.DATABASES["DB_NAME"] "DB_NAME"
  • Or remove database from Snowflake manually
  • Or use a different database name

Insufficient Privileges

Error:

Error: 001003 (42000): SQL access control error: Insufficient privileges

Solution:

  • Ensure provider uses SYSADMIN role
  • Verify user can assume SYSADMIN role
  • Check that roles exist for privilege grants

Role Not Found

Error:

Error: Role does not exist

Solution:

  • Ensure roles are created before databases
  • Verify role names in role_privileges match module outputs
  • Check that roles module is applied before database module
