Tables Module

The Tables module manages Snowflake tables with automatic privilege grant management.

Overview

The Tables module (modules/snowflake/tables) creates Snowflake tables within schemas and automatically grants privileges to specified roles. This module provides a complete solution for table management including column definitions and access control.

Module Location

modules/snowflake/tables/
├── main.tf        # Table resource definitions and grants
├── variables.tf   # Input variables
└── outputs.tf     # Output values

Usage

Basic Table

module "ALL_TABLES" {
  source = "./modules/snowflake/tables"
  
  providers = {
    snowflake = snowflake.snowflake_sysadmin
  }
  
  table_map = {
    "MY_TABLE" = {
      database_name = module.ALL_DATABASES.DATABASES.MY_DB.name
      schema_name   = module.ALL_SCHEMAS.SCHEMAS.MY_SCHEMA.name
      comment       = "My table"
      columns = [
        {
          name     = "ID"
          type     = "NUMBER(38,0)"
          nullable = false
        },
        {
          name = "NAME"
          type = "VARCHAR(255)"
        }
      ]
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.ANALYST.name
          privileges        = ["SELECT"]
        }
      ]
    }
  }
}

Resources Created

snowflake_table.TABLES

Creates tables using for_each over the table_map.

Properties:

  • name: Table name (from map key)
  • database: Parent database name
  • schema: Parent schema name
  • comment: Table comment
  • column: Dynamic blocks for each column

snowflake_grant_privileges_to_account_role.TABLE_GRANTS

Automatically grants table privileges to roles based on the role_privileges configuration.

Properties:

  • account_role_name: Role receiving privileges
  • privileges: List of privileges
  • on_schema_object.object_type: “TABLE”
  • on_schema_object.object_name: Fully qualified table name (database.schema.table)
  • with_grant_option: Whether grant includes grant option

Dependencies:

  • Depends on snowflake_table.TABLES to ensure tables exist before granting

Outputs

Data Types

Common Snowflake data types:

Numeric Types

  • NUMBER(p,s): Fixed-point number (precision, scale)
  • FLOAT, FLOAT4, FLOAT8: Floating-point numbers
  • INTEGER, INT, BIGINT, SMALLINT, TINYINT: Integer types

String Types

  • VARCHAR(n): Variable-length string (max 16MB)
  • CHAR(n): Fixed-length string
  • STRING: Alias for VARCHAR(16777216)
  • TEXT: Alias for VARCHAR(16777216)

Date/Time Types

  • DATE: Date only
  • TIME: Time only
  • TIMESTAMP_NTZ: Timestamp without timezone
  • TIMESTAMP_LTZ: Timestamp with local timezone
  • TIMESTAMP_TZ: Timestamp with timezone

Other Types

  • BOOLEAN: True/false values
  • VARIANT: Semi-structured data (JSON, etc.)
  • ARRAY: Array of values
  • OBJECT: Key-value pairs

Available Privileges

  • SELECT: Query the table
  • INSERT: Insert rows into the table
  • UPDATE: Update rows in the table
  • DELETE: Delete rows from the table
  • TRUNCATE: Truncate the table
  • REFERENCES: Reference the table in foreign keys
  • OWNERSHIP: Full ownership of the table
  • ALL PRIVILEGES: All of the above

Provider Requirements

The module requires the snowflake_sysadmin provider alias because:

  • Creating tables requires SYSADMIN privileges
  • Granting table privileges requires SYSADMIN privileges

Note: The provider configuration includes preview_features_enabled = ["snowflake_table_resource"] which may be required for table operations.

Dependencies

Internal Dependencies

  • snowflake_grant_privileges_to_account_role.TABLE_GRANTS depends on snowflake_table.TABLES

External Dependencies

  • Database Module: Tables must reference existing databases
  • Schema Module: Tables must reference existing schemas
  • Roles Module: Tables reference roles from the roles module for privilege grants

Best Practices

  1. Descriptive Names: Use clear, descriptive table and column names
  2. Comments: Always add comments explaining table and column purpose
  3. Nullable Columns: Explicitly set nullable for clarity
  4. Data Types: Choose appropriate data types for data
  5. Access Control: Grant appropriate privileges to roles
  6. Minimal Privileges: Grant only necessary privileges
  7. Database/Schema Reference: Always reference databases and schemas from modules

Troubleshooting

Table Already Exists

Error:

Error: 2003: Object already exists

Solution:

  • Import existing table: terraform import module.ALL_TABLES.snowflake_table.TABLES["TABLE_NAME"] "DB.SCHEMA.TABLE"
  • Or remove table from Snowflake manually
  • Or use a different table name

Schema Not Found

Error:

Error: Schema does not exist

Solution:

  • Ensure schema is created before table
  • Verify schema name is correct
  • Check that schema exists in Snowflake
  • Reference schema using module output: module.ALL_SCHEMAS.SCHEMAS.SCHEMA_NAME.name

Invalid Data Type

Error:

Error: Invalid data type

Solution:

  • Verify data type syntax is correct
  • Check that data type is supported by Snowflake
  • Review Snowflake data type documentation

Insufficient Privileges

Error:

Error: 001003 (42000): SQL access control error: Insufficient privileges

Solution:

  • Ensure provider uses SYSADMIN role
  • Verify user can assume SYSADMIN role
  • Check that user has necessary privileges
