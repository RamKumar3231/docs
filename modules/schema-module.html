Schema Module

The Schema module manages Snowflake database schemas with automatic privilege grant management.

Overview

The Schema module (modules/snowflake/schema) creates Snowflake schemas within databases and automatically grants privileges to specified roles. This module provides a complete solution for schema management including access control.

Module Location

modules/snowflake/schema/
├── main.tf        # Schema resource definitions and grants
├── variables.tf   # Input variables
└── outputs.tf     # Output values

Usage

Basic Schema

module "ALL_SCHEMAS" {
  source = "./modules/snowflake/schema"
  
  providers = {
    snowflake = snowflake.snowflake_sysadmin
  }
  
  schema_map = {
    "MY_SCHEMA" = {
      database_name       = module.ALL_DATABASES.DATABASES.MY_DB.name
      with_managed_access = false
      is_transient        = false
      comment             = "My schema"
      role_privileges = [
        {
          account_role_name = module.ALL_ROLES.ROLES.ANALYST.name
          privileges        = ["USAGE"]
        }
      ]
    }
  }
}

Schema with Managed Access

module "ALL_SCHEMAS" {
  source = "./modules/snowflake/schema"

    snowflake = snowflake.snowflake_sysadmin

}

Input Variables

Type: map(object)
Required: Yes
Description: Map of schema names to schema configurations

with_managed_accessboolYes-Enable managed access (object-level security)
is_transientboolYes-Whether schema is transient (no Time Travel)
commentstringNo""Comment describing the schema

Resources Created

snowflake_schema.SCHEMAS

Creates schemas using for_each over the schema_map.

Properties:

  • name: Schema name (from map key)
  • database: Parent database name
  • with_managed_access: Managed access setting
  • is_transient: Transient setting
  • comment: Schema comment
  • data_retention_time_in_days: Time Travel retention

snowflake_grant_privileges_to_account_role.SCHEMA_GRANT

Automatically grants schema privileges to roles based on the role_privileges configuration.

Properties:

  • account_role_name: Role receiving privileges
  • privileges: List of privileges
  • on_schema.schema_name: Fully qualified schema name (database.schema)
  • with_grant_option: Whether grant includes grant option

Dependencies:

  • Depends on snowflake_schema.SCHEMAS to ensure schemas exist before granting

Outputs

SCHEMAS

Type: map(object)

A map of all created schema resources, keyed by schema name.

Example Output:

{
  "MY_SCHEMA" = {
    name    = "MY_SCHEMA"
    database = "MY_DB"
    # ... other attributes
  }
}

Usage:

# Reference a schema in tables module
schema_name = module.ALL_SCHEMAS.SCHEMAS.MY_SCHEMA.name

Managed Access

When with_managed_access = true:

  • Enables object-level security
  • Requires explicit grants on objects (not inherited from schema)
  • More secure but requires more grant management
  • Recommended for sensitive data

Example:

schema_map = {
  "SECURE_SCHEMA" = {
    database_name       = module.ALL_DATABASES.DATABASES.MY_DB.name
    with_managed_access = true  # Enable managed access
    is_transient        = false
    # ... other configuration
  }
}

Transient Schemas

When is_transient = true:

  • No Time Travel (cannot query historical data)
  • No Fail-safe (no backup after Time Travel expires)
  • Lower storage costs
  • Use for temporary or non-critical data

Example:

schema_map = {
  "TEMP_SCHEMA" = {
    database_name = module.ALL_DATABASES.DATABASES.MY_DB.name
    is_transient  = true  # Transient schema
    # ... other configuration
  }
}

Data Retention

The data_retention_time_in_days property controls Time Travel retention for the schema:

  • Inherits from database if not specified
  • Can override database retention
  • Same retention limits as databases (1-90 days standard, up to 1 year enterprise)

Available Privileges

  • USAGE: Use the schema (required to access tables/views)
  • MODIFY: Modify schema settings
  • CREATE TABLE: Create tables in the schema
  • CREATE VIEW: Create views in the schema
  • CREATE FILE FORMAT: Create file formats
  • CREATE STAGE: Create stages
  • CREATE FUNCTION: Create functions
  • CREATE PROCEDURE: Create procedures
  • CREATE SEQUENCE: Create sequences
  • CREATE TASK: Create tasks
  • CREATE PIPE: Create pipes
  • CREATE STREAM: Create streams
  • MONITOR: View schema usage and metrics
  • OWNERSHIP: Full ownership of the schema
  • ALL PRIVILEGES: All of the above

Provider Requirements

The module requires the snowflake_sysadmin provider alias because:

  • Creating schemas requires SYSADMIN privileges
  • Granting schema privileges requires SYSADMIN privileges

Dependencies

Internal Dependencies

  • snowflake_grant_privileges_to_account_role.SCHEMA_GRANT depends on snowflake_schema.SCHEMAS

External Dependencies

  • Database Module: Schemas must reference existing databases
  • Roles Module: Schemas reference roles from the roles module for privilege grants

Best Practices

  1. Descriptive Names: Use clear, descriptive schema names
  2. Comments: Always add comments explaining schema purpose
  3. Managed Access: Use managed access for sensitive data
  4. Transient for Temp: Use transient for temporary data
  5. Access Control: Grant USAGE to roles that need access
  6. Minimal Privileges: Grant only necessary privileges
  7. Database Reference: Always reference databases from the database module

Troubleshooting

Schema Already Exists

Error:

Error: 2003: Object already exists

Solution:

  • Import existing schema: terraform import module.ALL_SCHEMAS.snowflake_schema.SCHEMAS["SCHEMA_NAME"] "DB.SCHEMA"
  • Or remove schema from Snowflake manually
  • Or use a different schema name

Database Not Found

Error:

Error: Database does not exist

Solution:

  • Ensure database is created before schema
  • Verify database name is correct
  • Check that database exists in Snowflake
  • Reference database using module output: module.ALL_DATABASES.DATABASES.DB_NAME.name

Insufficient Privileges

Error:

Error: 001003 (42000): SQL access control error: Insufficient privileges

Solution:

  • Ensure provider uses SYSADMIN role
  • Verify user can assume SYSADMIN role
  • Check that user has necessary privileges
